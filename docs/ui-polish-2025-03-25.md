# UI Polish — 2026-03-25

## Summary
A collection of UI improvements across the app: larger expand/collapse arrows, checkbox redesign, and project detail defaults.

## Expand/collapse arrows — app-wide
Bumped all `▸`/`▾` arrows from 9–11px to **18px** with `leading-none` for proper vertical alignment with adjacent labels. Widened hitboxes from `w-3`/`w-4` to `w-5` (and `w-6`→`w-7` on ProjectCard).

### Files changed
- `src/pages/DailyPlanner.tsx` — project expand, unscheduled expand, unfinished expand
- `src/pages/Projects.tsx` — project task expand
- `src/pages/WeeklyPlanner.tsx` — task list expand, notes expand
- `src/components/Sidebar.tsx` — shortcuts expand (also changed from `▾` with rotate(180°) to `▸` with rotate(90°) for consistency)
- `src/components/ProjectCard.tsx` — card expand

## Task checkbox redesign
Changed from empty box (not done) / green filled box (done) to:
- **Not done**: gray checkmark (`rgba(0,0,0,0.15)`) inside a subtle bordered box — indicates what clicking will do
- **Done**: green background (`#4a9e6e`) with white checkmark SVG

### File changed
- `src/components/TaskCard.tsx`

## Projects screen — completed tasks visible
- Expanded project cards now load **all tasks** including completed ones (removed 5-task limit, passed `includeDone: true`)
- Done tasks show with green checkmark + strikethrough, sorted below incomplete tasks
- Incomplete tasks show with gray checkmark

### File changed
- `src/pages/Projects.tsx`

## Project detail — show completed by default
- Changed `showDone` default from `false` to `true` so completed tasks appear below incomplete tasks on first load
- Toggle button still available to hide them

### File changed
- `src/pages/ProjectDetail.tsx`
